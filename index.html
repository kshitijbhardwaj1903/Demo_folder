Firstly we'll add our files using git add . and then now we'll go on github and create new empty repositary. Because we will decide ki hum kaunsi repositary mei apni files ko push karna chahte hai. And in most cases we will create a new empty repositary and usmei apne project ko push karenge. We can push our code in many styles. We can create a new repositary from command line and also add code from the code line. Another method is pushing the code to existing empty repositary, we'll use this method here. So we create a new empty repositary on github. Then we'll look for a link in https. Copy that, then write our command " git remote add origin (link)".
As mentioned in our imp_point file, we can use git remove -v to verify ki humari files kaunsi repositary pe push horhi hai. We use git branch -M main to change our master branch name to main branch name. (Agar master likha aye to, wrna directly hi main likha aega.) Now we can push our code in github using git push origin main. We create a new repo, push our project there using "git remote add origin" .... then using git push origin main.... and we can make small changes in our readme file or index file, and then use git add . then write git commit -m "name" then write git push origin main. Also, if we dont want to write git push origin main everytime, we can write 'git push -u origin main' once. And ab hum agar koi change kare, git add kare, then git commit kare, uske baad git push origin main likhne ki need nahi hai, we can only write git push. Using "git push -u origin main" ... hum origin ko set karrhe hai ki idhar hi sab add hoga types.
